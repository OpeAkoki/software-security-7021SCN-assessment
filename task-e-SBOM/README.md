# Task E — Compliance and Supply Chain

## What this task covers

This task has two parts. First, generating a Software Bill of Materials (SBOM) for the webapp and scanning it for known vulnerabilities in its dependencies. Second, mapping all the findings from the assessment back to the NIST SSDF (SP 800-218) to show which secure development practices were missing and what should have been done differently.

## SBOM Generation

The SBOM was generated using **Syft** via Docker, targeting the webapp directory from Task D. The output format is CycloneDX JSON, which is one of the two main SBOM standards alongside SPDX.

Command used:
```bash
docker run --rm -v $(pwd)/task-d-dynamictesting/webapp:/src anchore/syft /src -o cyclonedx-json > task-e-SBOM/sbom/sbom.cdx.json
```

The SBOM lists all Python packages present in the environment and their versions, giving a point-in-time record of the software components the application depends on.

## Vulnerability Scan

The SBOM was scanned using **Grype** via Docker to check all listed components against known CVE and GHSA databases.

Command used:
```bash
docker run --rm -v $(pwd)/task-e-SBOM/sbom:/sbom anchore/grype sbom:/sbom/sbom.cdx.json
```

Two vulnerabilities were found, both in `pip` version 24.0. Neither affects the running webapp directly but both should be remediated by upgrading pip to version 26.0 or later.

## SSDF Mapping

`ssdf-mapping.md` maps each finding from Tasks A through D to the relevant NIST SSDF SP 800-218 practice. It covers secure coding (PW.5), input validation (PW.6), code review (PW.7), testing (PW.8), and vulnerability management (RV.1 through RV.3).
